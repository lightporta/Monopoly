# 《仙境海岸·大富翁》游戏核心设计书

> **版本**：V1.0
> **生成日期**：2026-07-12
> **适用范围**：仙境海岸·大富翁网页版游戏产品设计

---

## 一、游戏概述

### 1.1 主题定位

本游戏以山东烟台文旅为主题，将烟台著名地标、美食、文化特色融入经典大富翁玩法，打造一款具有地域特色的休闲策略游戏。

### 1.2 平台定位

- **平台**：Web / H5 网页游戏
- **目标设备**：桌面浏览器、移动端浏览器
- **网络要求**：单机模式无需网络，联机模式需要网络连接

### 1.3 目标用户

- 休闲玩家：寻求轻松娱乐的普通用户
- 大富翁爱好者：喜欢经典桌游的玩家群体
- 烟台文旅兴趣群体：对烟台旅游文化感兴趣的用户

### 1.4 核心玩法循环

```
掷骰子 → 移动棋子 → 触发事件（购买/付租/抽卡/传送）→ 积累资产 → 达成胜利条件
```

---

## 二、设计目标

### 2.1 文旅融合

将烟台地域特色深度融入游戏：
- **地标地产**：芝罘湾广场、烟台山、蓬莱阁、养马岛等21处烟台地标
- **特色事件**：八仙过海传送、海昌鲸鲨馆再掷、烟台山灯塔定向传送
- **美食收集**：烟台焖子、蓬莱小面、海鲜疙瘩汤、鲅鱼水饺四种特色美食
- **铁三角胜利**：收集烟台山、蓬莱阁、养马岛三处核心地标

### 2.2 轻量化

- 单局时长：15-30分钟
- 规则简单：经典大富翁玩法，易于上手
- 体积小巧：单文件HTML，无需安装

### 2.3 支持AI对手

- 三种AI难度：新手导游、本地商人、仙境霸主
- AI自动决策：购买、建造、抵押等操作自动完成
- 支持混合模式：1-2名真人 + 1-3名AI

## 三、核心机制清单

### 3.1 掷骰移动

- 双骰子掷出，范围2-12
- 连续三次双数触发惩罚机制
- 棋子沿环形棋盘移动

### 3.2 地产经营

- 购买无主地产
- 建造房屋提升租金
- 抵押地产获取现金
- 赎回抵押地产

### 3.3 色块加成

- 四种色块分组：历史街区、海岸线、仙山、温泉
- 集齐条件触发租金加成
- 加成倍数：×1.5 ~ ×2.0

### 3.4 卡牌系统

- 机会卡：正面效果为主
- 命运卡：正负效果都有
- 美食卡：收集兑换奖励

### 3.5 传送与移动

- 烟台山灯塔：固定传送至蓬莱阁
- 八仙渡海口：传送至任意空地
- 海昌鲸鲨馆：再掷一次骰子
- 机场：传送至任意格子

### 3.6 美食收集

- 四种美食卡收集
- 集齐兑换奖励
- 免租券机制

### 3.7 双重胜利条件

- 破产胜利：使其他所有玩家破产
- 铁三角胜利：同时持有烟台山、蓬莱阁、养马岛

## 四、玩家体验曲线

### 4.1 开局（探索阶段）

- 玩家熟悉棋盘布局
- 开始购买第一处地产
- 体验基础掷骰和移动

### 4.2 中盘（博弈阶段）

- 色块收集竞争激烈
- 建筑升级提升收益
- 卡牌事件增加变数
- 玩家之间的租金博弈

### 4.3 终盘（决胜阶段）

- 资产差距拉大
- 破产风险增加
- 铁三角竞争白热化
- 最终胜负判定

## 五、差异点对比

与传统大富翁相比，本游戏有以下特色差异：

| 差异点 | 传统大富翁 | 仙境海岸·大富翁 |
|--------|-----------|----------------|
| 胜利条件 | 破产胜利 | 破产胜利 + 铁三角胜利 |
| 美食系统 | 无 | 美食卡收集与兑换 |
| 传送机制 | 无 | 八仙过海传送系统 |
| 定向传送 | 无 | 烟台山灯塔→蓬莱阁 |
| 再掷机会 | 无 | 海昌鲸鲨馆再玩一次 |

## 七、技术架构

### 7.1 技术栈

- **框架**：Vue 3 + TypeScript
- **状态管理**：Pinia
- **路由**：Vue Router
- **构建工具**：Vite
- **样式**：CSS3 + CSS Variables

### 7.2 架构分层

```
┌──────────────────────────────────────────────┐
│  Views (页面级)                              │  ← 路由 + 页面装配
├──────────────────────────────────────────────┤
│  Components (UI 组件)                        │  ← 纯展示，无业务
├──────────────────────────────────────────────┤
│  Store (状态管理)  ←→  Engine (游戏引擎)     │  ← 状态 + 纯逻辑
├──────────────────────────────────────────────┤
│  Data (JSON 静态数据)                        │  ← 数据源
└──────────────────────────────────────────────┘
```

### 7.3 核心原则

- **引擎与UI分离**：引擎层纯TypeScript，不依赖Vue，可独立测试
- **数据与逻辑分离**：所有可调参数进JSON，引擎代码只读不算
- **单一数据源**：每条数据只出现一次，避免冗余

## 八、功能模块

### 8.1 主菜单页

- 游戏标题与主题视觉
- 模式选择：与真人玩 / 与AI玩
- 玩家配置：人数、昵称、AI难度
- 游戏规则按钮：弹窗显示完整规则
- 退出游戏按钮

### 8.2 游戏主界面

- 棋盘区：36格环形布局
- 玩家面板：显示所有玩家状态
- 骰子区：掷骰子按钮与动画
- 操作按钮：建造、抵押、赎回、兑换美食
- 游戏日志：流式显示所有事件
- 顶部状态栏：退出按钮、当前玩家指示

### 8.3 结算页

- 获胜者展示
- 资产排名表格
- 再来一局按钮
- 返回主菜单按钮

### 8.4 弹窗系统

- 卡牌弹窗：机会卡、命运卡、美食卡
- 购买确认弹窗：购买或放弃地产
- 建造弹窗：选择建造房屋或旅馆
- 传送选择弹窗：选择目标格子
- 美食兑换弹窗：选择兑换选项
- 回合交接弹窗：PvP模式下提示换玩家
- 退出确认弹窗：确认退出游戏
- 胜利弹窗：显示获胜者

## 九、联机模式设计（未来扩展）

### 9.1 技术方案

- **实时通信**：WebSocket
- **服务端**：Node.js + ws库
- **房间管理**：内存Map管理房间状态
- **玩家身份**：playerId + playerName

### 9.2 功能特性

- 多房间并行：支持多个游戏房间同时进行
- 房间创建/加入：通过房间Key加入游戏
- 断线重连：30秒内自动重连
- 房主控制：房主可开始游戏、解散房间
